## Architecture

XSoftHub is an Astro 5 static site for a software development company. It supports two languages (English and Spanish) and uses pure Astro components (no React/Vue/Svelte). jQuery and Bootstrap are loaded from `public/assets/` for DOM manipulation and styling.

### Routing

File-based routing under `src/pages/[lang]/`. The root `index.astro` detects browser language and renders the appropriate language version. All content pages live under the `[lang]` dynamic segment:

- `/{lang}/` - Home page
- `/{lang}/services/{name}` - Service detail pages
- `/{lang}/saas/{name}` - SaaS product pages
- `/{lang}/contact` - Contact page

English URLs are also available without prefix (`prefixDefaultLocale: false` in astro config), but all internal links use the `/{lang}/` pattern. Each page exports `getStaticPaths()` from `src/i18n/utils.ts` to generate both `es` and `en` variants.

### i18n System

Custom translation system (not a library):

1. **Translation files**: JSON files in `src/i18n/components/` and `src/i18n/pages/`, structured as `{ "es": {...}, "en": {...} }`
2. **Aggregation**: `src/i18n/ui.ts` imports all JSON files and merges them into a single `ui` object per language
3. **Usage**: Components call `useTranslations(lang)` which returns a `t()` function for key lookup
4. **Language resolution**: `resolveLanguage(params, request)` checks URL param first, then browser `Accept-Language` header, then defaults to `en`

When adding a new translatable string, add keys to both `es` and `en` in the relevant JSON file. If it's a new file, import and spread it into `ui.ts`.

### Component Patterns

- All components receive a `lang: SupportedLanguage` prop (defined in `src/types/index.ts`)
- Section components live in `src/components/sections/` and compose into pages
- Icon components are organized in `src/components/icons/{ui,contact,social}/`
- `ServicePage.astro` is a reusable template for service detail pages with a slot for extra content
- Layout wrapping (SEO meta, header, footer) is handled by `src/layouts/Layout.astro`

### Data Flow for Services

Service listings are generated by `src/utils/data-store.ts` which uses `useTranslations()` to produce localized service arrays with title, image path, and link. These are consumed by section components.

### Static Assets

All static assets (CSS, JS, images, fonts) are in `public/assets/`. Image path constants are in `src/utils/constants.ts`. External libraries (jQuery, Bootstrap, GSAP, Slick, WOW.js) are loaded from `public/assets/js/` and `public/assets/css/`.

### TypeScript

Uses Astro's strict TypeScript config. Shared types are in `src/types/index.ts`. The base interface `ComponentProps` provides the `lang` prop that nearly all components extend.
